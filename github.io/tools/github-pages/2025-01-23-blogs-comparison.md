## 개발용 블로그 비교

개발용 블로그를 이것 저것 찾아보다가 다시 github pages로 돌아온 이유

> 결론 : github pages의 단점에도 불구하고, 다른 플랫폼에서 제공하지 못하는 강력한 장점이 존재함

#### 각 플랫폼 사용시 느낀 점

- tistory
  - 장점 : 접근성이 좋음
  - 단점 : markdown이 제대로 지원되지 않음.
- velog
  - 장점 : 깔끔함, markdown 지원이 잘됨. 한글화, 개발인맥, 쉬운 이미지 업로드
  - 단점 : mermaid 미지원, 커스터마이징 어려움, 내 수익이 아닌 광고가 마음대로 달릴 수 있음, git 연동이 되지 않아 글을 되돌릴 수 없음
- hashnode
  - 장점 : mermaid 지원, 깔끔함, git 연동, 쉬운 이미지 업로드
  - 단점 : raw markdown 에디터가 사용이 불편함. 한글 미지원, 커스터마이징 어려움
- github pages
  - 장점 : mermaid 지원, git 연동, 커스터마이징, 파일 및 폴더 형태의 글관리
  - 단점 : 어려운 이미지 업로드, markdown 변환시 가끔 에러발생하여 포인트 찾는데 시간이 오래 걸림, 로그인을 통한 비공개 기능의 부재

#### 경험상 개발용 블로그 플랫폼이 해원해야할 부분

- markdown 지원
- mermaid 지원
- 글 삭제 후 복구 가능 (git 연동)
- 커스터마이징
- 폴더 및 파일 형태의 글 관리
