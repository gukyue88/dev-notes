# Part 1. 소프트웨어 3.0

## 프롬프트 엔지니어링

- 프로그래밍 == 컴퓨터와의 상호작용 방법
- 프롬프트 엔지니어링 == 컴퓨터와의 새로운 상호작용 방법 == 자연어로 컴퓨터와 상호작용하는 방법
- 소프트웨어 3.0 : 기존 소프트웨어 방법에 프롬프트 엔지니어링이 더해진 방식
  - 더 넓고 복잡한 영역을 다루는 프로그램 개발이 가능해짐
  - 결정론적(Code)과 통계적(ML) 프로그램 양쪽 모두 개발 속도를 빠르게 함
  - 더 많은 사람이 프로그래머가 될 수 있음
- 프롬프트 엔지니어링은 머신러닝 모델링 방법과 개발 방법을 혼합해 놓은 것과 비슷함
- 머신러닝 모델링 방법과 개발 방법론 둘 다 이해하고 활용할 수 있어야 높은 수준의 프롬프트 엔지니어링을 할 수 있음
- 프롬프트 엔지니어링은 실험적 방법론에 기반하여, 한 번 그럴듯한 결과를 내는 것이 아닌, 원하는 결과를 정확히 의도한대로 항상 일관성 있게 내도록 만드는 것임
