## Chapter 4. 어떻게 가능해졌나

#### Autoregressive Model

- 앞에 단어들을 보고 다음에 올 단어를 예측하는 형태로 동작함
- 그리고 그 단어를 합쳐서 다음에 올 단어를 다시 예측하는 방식임
- 언어모델의 크기를 키웠더니 점점 더 기능이 좋아졌더라 하는게 gpt임

#### 코드 데이터 학습

- 코드 데이터를 학습함으로 인해 추론 능력이 향상되고, 정형 데이터를 생성할 수 있게됨
- gpt 3.5의 경우, 코드를 학습한걸 기반으로 일반적인 언어를 학습하여 이런 능력을 갖출 수 있게 됐다고 함

#### Instruction tuning

- 명령/지시로 표현된 텍스트를 이해햐여, 지시한 작업을 수행
- 인스트럭션 튜닝을 통해 사람의 명령을 이해해서, 그 명령을 가지고 그 다음 텍스트를 생성하는 방식이 가능해짐

#### Multimodal

- 텍스트외 데이터도 같이 학습하여 더 강력한 기능을 갖게 됨

## Chapter 5. 이것이 왜 그렇게 중요한가

#### 데이터 인터페이스로서의 LLM

- 비정형과 정형 데이터의 양방향 인터프리터 역할을 할 수 있음
- 과거에는 정형 데이터가 아닌 비정형데이터(자연어어는 분석하기 어려웠지만, 이제는 LLM으로 분석 작업을 맡길 수가 있음
- 데이터의 학습이나 전처리 과정이 필요 없이, 데이터를 탐고하여 분석하고 나아가 요구사항에 따른 적합한 액션(예. 그래프 그리기, 인사이트 등)까지 수행할 수 있음

#### ML Workflow 혁신

- 전통적인 ML : 프로젝트 설정 -> 데이터 모으기 -> 데이터 모델링 -> 디플로이
  - 각 단계가 1~3 개월씩 걸림
- Prompt-based ML : 프로젝트 설정 -> 프롬프트 엔지니어링 -> 디플로이
  - 각 단계가 몇 분, 몇 일 정도면 끝이남

## Chapter 6. 파인튜닝 vs 프롬프트 엔지니어링

- 빅모델을 사용하면 파인튜닝과 프롬프트 엔지니어링의 성능차이가 크지 않음
- 모델링 혹은 파인튜닝이 필요한 경우
  - 숫자를 예측하는 선형 회귀 문제
  - 대량의 로그성 데이터의 실시간 처리
  - 특수 목적의 매우 높은 정밀도를 요구하는 문제
  - 데이터의 최신성이 중요하지 않은 경우
  - 데이터 보안이 매우 중요한 경우

## Chapter 7. 새로운 기회 찾기

- 생성 AI와 llm을 통한 기회는 어디에 있을까?

#### 생성모델의 한계

- 아주 정밀하게 생성하지는 못함
- 답변이 올바른지 확인할 수 없음
- 학습 시점의 정보만 사용함
- 정보의 정확성, 윤리성 확인
- 저작권 이슈 고려
- 이런 것들은 gpt에게 툴을 쥐어주거나, 최신 정보를 컨텍스트로 주입해줌으로서 해결할 수도 있음
- 또, 모델이 업데이트 되면서 점점 이런 문제도 줄어들고 있음

#### 창의력의 종말?

- 창의력 == 무엇을 표현할지를 생각하는 것
- 더 많은 사람들이 AI라는 도구를 가지고 자신의 생각을 더 쉽게 표현할 수 있게 된 것

#### 기회

- 컴퓨터가 말을 알아듣기 시작하여 할 수 없었던 일을 할 수 있게 되고 완전히 새로운 방식으로 생활하게 될 것으로 예상됨

#### 도입현황

- 모든 회사가 도입하고 싶어 하지만, 모두 뭘 해야 할지를 모르는 상태
- 경영진 레벨에서는 기회와 위기감을 가짐
  - 수천억짜리 기술의 비용이 극도로 저렴해져 누구나 쉽게 도입 할 수 있게 됨
- 예전에는 해결하지 못했던 사람들의 문제를 푸는 것에 기회가 있을 것임
